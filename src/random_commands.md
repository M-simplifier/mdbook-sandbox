# Random Command Sequences

PBTで威力を発揮するのが「ランダムなコマンド列」です。単一操作では見えない組み合わせバグを、低コストで探索できます。

## 基本方針

- コマンド列の長さはサイズに依存させる
- 失敗時に短く縮むように設計する
- Insert/Normalの切替を必ず含める

## 実装のポイント

- `genCommandSeq` で `vectorOf` を使う
- 長さを `min 30 (n + 5)` のように制限し、テストの重さを抑える
- コマンドの分布は後から調整しやすいように一箇所で定義する

## よくある失敗例

- カーソル位置が行末を越える
- Insert中に連続でBackspaceした時の挙動が壊れる
- 行の境界で左右移動したときに行番号がずれる
- 改行後にカーソルが不正な位置に残る
- 行削除後にカーソルが消える

これらは、単発テストでは見落としやすい典型例です。
